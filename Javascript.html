Javascript Basics

Variables

Variables are used to store the data in JavaScript

  • var
    1. Scope:Global scope
    2. Hoisting: Variables declared with var are hoisted to the top of the function or global scope
    3. Re-assigningAssigned value can be changed after the declaration.
    4. Re-declaration:Can be re-declared with in the scope.
    5. Intilization resource:No need to be initialized when declared.
  • let
    1. Scope:Within block
    2. Hoisting: Variables declared remains in the block scope
    3. Re-assigningAssigned value can be changed after the declaration.
    4. Re-declaration:Cannot be re-declared with in the same scope.
    5. Intilization resource:No need to be initialized when declared.
  • const
    1. Scope:Within block
    2. Hoisting: Variables declared remains in the block scope
    3. Re-assigningAssigned value cannot be changed after the declaration.
    4. Re-declaration:Cannot be re-declared with in the same scope.
    5. Intilization resource:Must be initialized when declared.

Map

A Map holds key-value pairs where the keys can be any datatype. A Map remembers the original insertion order of the keys.

const fruits = new Map([["apples",500],["banana",200],["oranges"],300]]); 
const fruits = new Map();
fruits.set("apples",500);
fruits.set("banana",200);
fruits.set("oranges",300);

console.log(fruits.get("apples")) - Output: 500

Functions

A function is a block of code that performs a specific task.

Types of Functions
  • Function declaration
    • function greet() {
console.log("Hello, world!");
}
                              
greet(); // Output: Hello, world!
  • Function Expression
    • const greet = function() {
console.log("Hello, world!");
};
                              
greet(); // Output: Hello, world!
  • Arrow Functions
    • const greet = () => {
console.log("Hello, world!");
};
                              
greet(); // Output: Hello, world!
  • Anonymous Functions
    • setTimeout(function() {
console.log("This is an anonymous function");
}, 1000);
  • Higer-order Functions
    • const numbers = [1, 2, 3, 4];
const doubled = numbers.map(n => n * 2);
console.log(doubled); // Output: [2, 4, 6, 8]
  • Callback Functions
    • function fetchData(callback) {
setTimeout(() => {
    callback("Data fetched!");
}, 1000);
}
                              
fetchData(function(message) {
console.log(message); // Output: Data fetched!
});
  • IIFE (Immediately Invoked Function Expression)
    • (function() {
console.log("This function runs immediately!");
})();

    Or

    (() => {
console.log("This function runs immediately!");
})();
  • Recursive Functions
    • function factorial(n) {
if (n <= 1) return 1;
return n * factorial(n - 1);
}
                              
console.log(factorial(5)); // Output: 120
  • Generator Functions
    • function* generateNumbers() {
yield 1;
yield 2;
yield 3;
}
                              
const generator = generateNumbers();
console.log(generator.next().value); // Output: 1
console.log(generator.next().value); // Output: 2
console.log(generator.next().value); // Output: 3
  • Async Functions
    • async function fetchData() {
const response = await fetch("https://api.example.com");
const data = await response.json();
console.log(data);
}
                              
fetchData();
  • Object Methods
    • const person = {
name: "Alice",
greet: function() {
console.log(`Hello, ${this.name}`);
}
};
                              
person.greet(); // Output: Hello, Alice
  • Constructor Functions
    • function Person(name, age) {
this.name = name;
this.age = age;
}
                              
const person1 = new Person("Alice", 30);
console.log(person1.name); // Output: Alice

Events

Events - event can be something the browser does, or something a user does.

  • onchange - fires when the value of an element changes.
  • onclick - fires when an element is clicked.
  • onmouseover - fires when the mouse moves over an element.
  • onmouseout - fires when the mouse moves out of an element.
  • onkeydown - fires when a user presses a key.
  • onkeyup - fires when a user releases a key.
  • onmousedown - fires when a user presses down on an element.
  • onmouseup - fires when a user releases a mouse button.
  • onmousemove - fires when the mouse moves over an element.
  • onload - fires when a page has finished loading.
