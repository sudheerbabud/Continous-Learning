Introduction of Springboot

Spring Boot is advance version of Spring, where we don't have to do much configuration it has own capability to do on its own

Spring Boot is an open-source framework that makes it easy to create stand-alone, production-ready Spring applications.

Spring Boot provides a few key features:

  • Convenient dependency management
  • Embedded web servers (Tomcat, Jetty, Undertow)
  • Auto-configuration for various application needs
  • Starter projects for common use cases

Spring Boot Annotations

  • @SpringBootApplication – Combines @Configuration, @EnableAutoConfiguration, and @ComponentScan.
  • @EnableAutoConfiguration – Enables Spring Boot's auto-configuration.
  • @ComponentScan – Scans for Spring components automatically.
  • @ConfigurationProperties – Binds external properties to Java objects.
  • @Autowired – Injects dependencies into a bean.
  • @RestController – Indicates that a class is a controller that returns HTTP responses.
  • @RequestMapping – Maps HTTP requests to controller methods.
  • @GetMapping – Maps HTTP GET requests to controller methods.
  • @PostMapping – Maps HTTP POST requests to controller methods.
  • @PutMapping – Maps HTTP PUT requests to controller methods.
  • @DeleteMapping – Maps HTTP DELETE requests to controller methods.
  • @PathVariable – Extracts a value from the request path and binds it to a method parameter.
  • @RequestBody – Binds the request body to a method parameter.
  • @RequestParam – Extracts a value from the request query string and binds it to a method parameter.
  • @Service – Indicates that a class is a service, which is a type of Spring bean.
  • @Repository – Indicates that a class is a Spring data repository.

Spring Boot JPA

it is a API ... JPA (Java Persistence API) is used to simplify database interactions in Java applications. Instead of writing complex SQL queries and JDBC code, JPA allows developers to work with Java objects (entities) and let the ORM (like Hibernate) handle the database operations.

It uses Hibernate to interact to databases

Hibernate

  • Hibernate is a fully functional ORM framework that implements the JPA specification.
  • Hibernate uses an object-relational mapping (ORM) approach to map Java objects to relational databases.
  • Hibernate provides a simple and powerful way to access and manipulate data from relational databases.
  • Hibernate supports various databases, including MySQL, PostgreSQL, Oracle, and SQL Server.
  • Hibernate internally uses JDBC to interact with the database.

Postman Tool

Postman tool is used to POST, GET, PUT, DELETE that means we can send the data to database create, read, update and delete them by using postman tool

  • when we have post the data to database then we use post method and we have to select body and raw and select JSON options .. Springboot uses only JSON
  • when we have get the data from database then we use get method and we have to select params and select query parameters .. Springboot uses only query parameters
  • when we have put the data to database then we use put method and we have to select body and raw and select JSON options .. Springboot uses only JSON
  • when we have delete the data from database then we use delete method and we have to select params and select query parameters .. Springboot uses only query parameters

Response Status Codes

  • 200 OK: The request was successful.
  • 201 Created: The request has been fulfilled and a new resource has been created.
  • 204 No Content: The server successfully processed the request but is not returning any content.
  • 400 Bad Request: The server cannot or will not process the request due to an apparent client error.
  • 401 Unauthorized: The request requires user authentication.
  • 403 Forbidden: The server understood the request but refuses to authorize it.
  • 404 Not Found: The server did not find the requested resource.
  • 500 Internal Server Error: The server encountered an unexpected condition that prevented it from fulfilling the request.

Spring Boot Actuator

It is a dependency that is added in the POM.XML file.

It provides various endpoints for monitoring and managing the application.

Some of the endpoints provided by Actuator are:

  • /actuator/health
  • /actuator/info
  • /actuator/metrics
