JDBC - Architecture

JDBC Drivers is classified into 4 Types

  1. JDBC - ODBC Bridge Driver - thick driver --> obselete(Depricated)
  2. Native-API Driver - thick driver --> obselete
  3. Network Protocol Driver - thin driver
  4. Native Driver - thin driver
Steps to Connect Database
  1. Loading the Driver
  2. Connection establsih
  3. Statement
  4. ResultSet
  5. Close Connection
Statements in JDBC
  1. statement() - Used for static SQL queries.
  2. PrepatedStatement() - Used for precompiled SQL queries, with parameterized queries to avoid SQL injection.
  3. CallableStatement() - Used to execute stored procedures.
Executing Queries in JDBC
  1. executeUpdate() - used for insert, update, delete
  2. executeQuery() - used for select - returns ResultSet
  3. execute() - used for DDL - returns boolean

Singleton Design Pattern

Singleton - one instance or object for a class

Pattern Types

  • Early Instantiation
  • Lazy Instantiation

Single Design Pattern consists of

  • static data
  • static method
  • private constructor
Early Instantiation
  • We are creating the object before the request.
  • private static final Employee e = new Employee();
Lazy Instantiation
  • We are creating the object after one request.
    • We must response to that request
  • We are creating the object after more requests.
    • We must response to that request based on any logic like first came or other. by using synchronized keyword

Java Server Page

  • JSP stands for Java Server Page.
  • It is a technology for the development of dynamic web pages.
  • It is a combination of HTML and Java code.
  • It is used to separate the presentation layer from the business logic layer.

JSP Tags

  1. Scripting Tags
  2. Directives Tags
  3. Actions Tags
  4. Custom Tags

JSP Life Cycle

There are 7 phases in JSP life cycle:

  1. Page Translation --> The JSP page is parsed and a Java file containing the corresponding Servlet is created.
  2. Compilation --> The Java file is compiled into a class file.
  3. Load class:- The compiled Servlet class is loaded.
  4. Instantiation:- An instance of the Servlet class is created.
  5. Call jspInit():- This method is called before any other method to allow initialization. It is called only once.
  6. Call _jspService():- This method is called for each request.
  7. Call jspDestroy():- This method is called when the Servlet container decides to take the Servlet out of service. It is called only once.

JSP Tags

  1. Scripting Tags
    • Expression Tag --> <%= code here %>
    • Declaration Tag --> <%! code here %>
    • Scriptlet Tag --> <% code here %>
  2. Directive Tags
    • Page Tag
      • Provides instructions to web container which are relevant to current JSP.
      • It specifies the page specific attributes.
      • <%@ page import="java.io.* %>
      • <%@ page contentType=application/msword %>
    • Include Tag
      • Location can be hidden from direct user interaction.
      • File extension does not matter because resource is copied into generated servlet before compilation.
      • Must be a file from local machine.
      • To combine or include the content of other resource into the current JSP page.
      • Another resource can be JSP, HTML or a text file. code reusability.
      • <%@ include file="header.html" %>
      • <%@ include file="/WEB-INF/jsp-inc/header.jsp" %>
    • Taglib Tag
      • To define a tag library that defines many tags.
      • <@% taglib uri="tld-loc" prefix="pfx" %>
      • taglib-uri - path identifier used in JSP taglib directive.
      • taglib-location - context-relative location of tld.
  3. Action Tags
    • JSP specification provides the action tags to control: Behaviour of the servlet engine, Control page flow Dynamically insert a file Reuse JavaBeans components etc.
    • jsp:actionName attributeName="attributeValue"/
    • jsp:include: combine resources at runtime.
    • jsp:forward: conditionally display a page
    • jsp:param: pass arguments HTML forms (used with forward and include).
    • jsp:plugin: write/ combines code for applets.
    • jsp:setProperty:sets bean value of property.
    • jsp:getProperty: gets bean value of property.
  4. Custom Tags
    • Custom tags are user-defined action tags.
    • A tag handler is associated with each tag for implementing the operations.
    • prefix : suffix attribute = "value"/>

Page Directive Attributes

  • Import → To import interface, classes or packages
  • Session → For checking whether JSP page is in HTTP session or not. It can have true or false value.
  • Buffer → For defining the buffer size.
  • Autoflush → Controls the performance of the buffer.
  • contentType
  • isErrorPage → Specifies the current JSP page can be used as error page.
  • errorPage
  • isThreadSafe → Specifies whether this page supports multithreading or not.

taglib Directive Attributes

  • uri --> Specifies the location of the tag library.
  • prefix --> Informs the web container that is used for custom actions.
