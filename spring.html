Dependency Injection

Dependency Injection (DI) is a core concept in the Spring Framework that allows objects to be loosely coupled by injecting their dependencies rather than creating them directly within the class. This improves maintainability, testability, and scalability of the application.

In Spring, we can use the following three ways to inject dependencies:

  • Constructor Injection --> By using constructor
  • Setter Injection --> By using setter method

Spring MVC

Spring MVC is a part of Spring Framework that provides a flexible and powerful ways to interact with Spring MVC.

Spring MVC is a model-view-controller (MVC) framework that enables developers to build web applications by separating the application's code into distinct components.

Spring MVC provides the following features:

  • Model-View-Controller (MVC) architecture
  • URL mapping
  • View resolution
  • Data binding
  • Exception handling

To create a Spring MVC project, follow these steps:

  1. Create a new Spring Boot project using the Spring Initializr.
  2. Add the 'Web' and 'MVC' dependencies to your pom.xml file.
  3. Create a controller class (e.g., HomeController.java) and define the required methods for handling HTTP requests.
  4. Create a view template (e.g., index.html) in the 'templates' directory.
  5. Define the URL mapping in the Spring Boot configuration class (e.g., WebMvcConfig.java) using the @ControllerAdvice and @RequestMapping annotations.

Spring Annotations

Spring provides various annotations that can be used to configure and enhance the behavior of Spring components.

Some commonly used Spring annotations include:

Spring Core Annotations

  • @Component – Marks a class as a Spring-managed component.
  • @Service – Defines a service layer component.
  • @Repository – Marks a class as a repository (DAO layer).
  • @Autowired – Injects dependencies automatically.
  • @Qualifier – Specifies which bean to inject when multiple beans exist.
  • @Primary – Gives preference to a specific bean when multiple exist.
  • @Bean – Defines a bean in a Java configuration class.
  • @Configuration – Indicates a Java-based configuration class.
  • @PropertySource – Loads properties from external files.
  • @Value – Injects property values from configuration files.
  • @Scope – Specifies the bean scope (singleton, prototype, etc.).
  • @Lazy – Delays the bean initialization.
  • @PostConstruct – Executes a method after bean initialization.
  • @PreDestroy – Executes a method before bean destruction.

Spring MVC Annotations

  • @Controller – Defines a Spring MVC controller.
  • @RestController – Combines @Controller and @ResponseBody for REST APIs.
  • @RequestMapping – Maps web requests to handler methods.
  • @GetMapping – Handles HTTP GET requests.
  • @PostMapping – Handles HTTP POST requests.
  • @PutMapping – Handles HTTP PUT requests.
  • @DeleteMapping – Handles HTTP DELETE requests.
  • @PatchMapping – Handles HTTP PATCH requests.
  • @ResponseBody – Sends data in the response instead of a view.
  • @RequestParam – Extracts query parameters.
  • @PathVariable – Extracts values from the URL path.
  • @RequestBody – Binds request body data to a method parameter.
  • @ResponseStatus – Defines the HTTP status code for a response.
  • @ExceptionHandler – Handles exceptions in controllers.

Spring Transaction Management Annotations

  • @Transactional – Manages transactions automatically.
  • @EnableTransactionManagement – Enables Spring's transaction management.

Spring Scheduling & Asynchronous Execution Annotations

  • @EnableScheduling – Enables scheduled tasks.
  • @Scheduled – Defines a scheduled task.
  • @Async – Enables asynchronous execution.
  • @EnableAsync – Enables Spring’s async processing.

Spring Caching Annotations

  • @EnableCaching – Enables caching support.
  • @Cacheable – Caches the result of a method.
  • @CacheEvict – Removes specific cache entries.

Project Requirements

To create a web application using Spring MVC, follow these files requirements:

  • pom.xml --> Maven configuration file
  • web.xml --> Deployment descriptor file
  • name-servlet.xml --> Main Deployment descriptor file
  • Model --> Data model classes
  • Controller --> Controller classes
  • DAO --> Data Access Object classes
  • JSP's --> View files
