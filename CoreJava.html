Introduction of Core Java

Java is a popular programming language that was developed by Sun Microsystems in 1995. It is a statically-typed, object-oriented programming language. Java is designed to be easy to learn and use, making it suitable for beginners and experienced developers alike.

Java has several key features that make it a popular choice for developing large-scale applications:

  • Simple
  • Object-oriented programming
  • Platform Independent
  • Secure
  • Robust
  • Multithreading
  • Dynamic

    Access Modifiers

    1. public
      • We can access the data members within the project.
    2. private
      • We can access the data members within the class Only.
    3. protected
      • We can access the data members within the package and class that import parent class.
    4. default
      • We can access the data members within the package.
    5. static
      • It is used to access the variables, methods, and blocks without creating an object of the class.
    6. synchronized
      • In multithreading, it is used to wait the second thread until the first thread finishes its execution.
    7. strictfp
      • It is used to get the get the correct floating points each time.
    8. abstract
      • When we declare a class as abstract, we cannot create an object of that class.
      • when we declare a method as abstract, we cannot implement that method.
    9. volatile
      • Volatile means keep on changing.
    10. transient
      • transient is a keyword to secure data when the objects are serailzed.
    11. native
      • it provides the implementation to java from another language like c,c++.
    12. final
      • when we declare a class as final, we cannot extends that class.
      • when we declare a method as final, we cannot override that method.
      • when we declare a variable as final, we cannot change the value of that variable.

    Object-Oriented Programming system (OOPS)

    Main concepts of OOPS:

    1. Encapsulation --> security and modularity
    2. Abstraction --> security
    3. Inheritance --> reusability
    4. Polymorphism --> flexibility

    Encapsulation

    Encapsulation is a mechanism of bundling data and methods that work on that data within a single unit.

    Abstraction

    Abstraction is a process of hiding implementation and showing the specifications.

    abstract

    when abstract is given to class then it must only have one abstract method (without any implementation) and any number of default or other methods.

    we cannot create an object of abstract class

    Interface

    Interface is a collection of abstract methods

    There are 3 types of Interfaces:

    1. Normal Interface --> all types of methods like static, private etc..
    2. Marker Interface --> no methods -->only specialization
    3. Functional Interface -->one abstract methods and other methods are default/static

    Inheritance

    Inheritance is a mechanism by which one class acquires the properties (methods and fields) of another class.

    We can achieve inheritance by using extends and new keywords.

    extends --> is-a relationship

    new --> has-a relationship

    Types of Inheritance:
    1. Single Inheritance --> only one parent class
    2. multiple Inheritance --> more than one parent
    3. Multilevel Inheritance --> multiple levels of inheritance
    4. Hierarchical Inheritance --> multiple child classes
    5. Hybrid Inheritance --> combination of single, multiple, and hierarchical inheritance
    6. cyclic Inheritance --> parent and child are same --> it is not possible

    Polymorphism

    Polymorphism is a feature in which a variable of a subclass can refer to an object of its superclass.

    Types of Polymorphism:

    • Method Overloading --> same method name with different parameters in same class
    • Method Overriding --> same method name with same parameters in different classes

    Some methods of string
    1. length() --> returns length of string

    Exception Handling

    Exception --> is an event that occurs during the execution of a program due to bad logic or unexpected input.

    Types of Exceptions:

    1. Checked Exceptions --> the exceptions that are checked by the compiler are checked exceptions.
      • IOException
      • SQLException
      • ClassNotFoundException
      • FileNotFoundException
      • Interrupted Exception
    2. Unchecked Exceptions --> the exceptions that are not checked by the compiler are unchecked exceptions.
      • ArithmeticException
      • ArrayIndexOutOfBoundsException
      • NullPointerException
      • ClassCastException
      • IllegalArgumentException
      • StringIndexOutofBoundsException
      • NoSuchElementException
      • ConcurrentModificationException
      • IllegalThreadstateException
      Error

      Errors --> errors occurs due to resources problems like out of memory, stack overflow etc.

      • OutOfMemoryError
      • StackOverflowError

      We can handle exceptions using try-catch-finally blocks.

      1. try block --> we can write the problematic code in this block.
      2. catch block --> whatever exception occurs in the try block, catch block will catch it.
      3. finally block --> this block will execute whether exception occurs or not.
      4. throw --> we can throw an exception explicitly.
      5. throws --> exceptions are thrown to satisfy the compiler

      We have 2 ways to handle exceptions:

      • by using try-catch-finally blocks --> For Graceful Termination of Program
      • by using throw keyword --> Gives Abnormal Termination of Program
      Try-catch-finally is the best way to handle exceptions

    Collections

    Collection is parent interface of list, set, queue, dequeue, sortedSet, navigableSet.

    Classes of collections: ArrayList, LinkedList, Vector, Stack, PriorityQueue, HashSet, TreeSet, LinkedHashSet etc..

    Methods of Collection
    • add(Object obj) --> adds the object to the collection
    • remove(Object obj) --> removes the object from the collection
    • contains(Object obj) --> checks whether the collection contains the object
    • size() --> returns the number of elements in the collection
    • clear() --> removes all elements from the collection
    • isEmpty() --> checks whether the collection is empty
    List

    List is an interface that extends Collection and provides additional methods to handle elements in specific order.

    Classes of list: ArrayList, LinkedList, Vector, Stack, CopyOnWriteArrayList etc..

    • List allows duplicate elements.
    • List allows homogenous and heterogeneous elements.
    • List allows null values any number of times.
    • Insertion order preserved.
    Methods of List
    • add (E e) --> adds the element at the end of the list
    • remove(int index) --> removes the element at the given index
    • contains (Object obj) --> checks whether the list contains the object
    • indexOf(Object obj) --> returns the index of the first occurrence of the object in the list
    • lastIndexOf(Object obj) --> returns the index of the last occurrence of the object in the list
    • get(int index) --> returns the element at the given index
    • set(int index, E e) --> replaces the element at the given index with the new element
    • addFirst(E e) --> adds the element at the beginning of the list
    • addLast(E e) --> adds the element at the end of the list
    • removeFirst() --> removes the element at the beginning of the list
    • removeLast() --> removes the element at the end of the list
    • iterator() --> returns an iterator over the elements in the list
    • listIterator() --> returns a list iterator over the elements in the list
    • subList(int fromIndex, int toIndex) --> returns a view of the portion of this list between the specified fromIndex, inclusive, and toIndex, exclusive
    • toArray() --> returns an array containing all the elements in the list
    • sort(Comparator c) --> sorts the elements in the list in ascending order according to the natural ordering of their corresponding elements, using the provided comparator
    • hashCode () --> returns the hash code value for this list
    • equals(Object obj) --> compares this list with the specified object for equality
    • toString() --> returns a string representation of the list
    • clone() --> creates and returns a shallow copy of this list
    • remove() --> removes the first occurrence of the specified element from this list, if it is present
    • size() --> returns the number of elements in the list
    Queue

    Queue is an interface that extends Collection and adds additional methods to handle elements in FIFO (First-In-First-Out) order.

    Classes of queue: PriorityQueue, LinkedList, ArrayDeque

    Methods of Queue
    • add(E e) --> adds the element at the end of the queue
    • remove() --> removes and returns the element at the front of the queue
    • poll() --> removes and returns the element at the front of the queue, or returns null if the queue is empty
    • element() --> returns the element at the front of the queue without removing it
    • offer(E e) --> adds the element at the end of the queue, returns true if successful, false if the queue is full
    • peek() --> returns the element at the front of the queue without removing it, or returns null if the queue is empty
    • size() --> returns the number of elements in the queue
    • isEmpty() --> returns true if the queue is empty, false otherwise
    Deque

    Deque is an interface that extends Queue and adds additional methods to handle elements in both ends (front and rear) of the queue. it is also called LILO (Last-In-Last-Out)order.

    Classes of deque: ArrayDeque, LinkedList

    Methods of Deque
    • addFirst(E e) --> adds the element at the beginning of the deque
    • addLast(E e) --> adds the element at the end of the deque
    • removeFirst() --> removes and returns the element at the beginning of the deque
    • removeLast() --> removes and returns the element at the end of the deque
    • pollFirst() --> removes and returns the element at the beginning of the deque, or returns null if the deque is empty
    • pollLast() --> removes and returns the element at the end of the deque, or returns null if the deque is empty
    • getFirst() --> returns the element at the beginning of the deque without removing it, or returns null if the deque is empty
    • getLast() --> returns the element at the end of the deque without removing it, or returns null if the deque is empty
    • offerFirst(E e) --> adds the element at the beginning of the deque, returns true if successful, false if the deque is full
    • offerLast(E e) --> adds the element at the end of the deque, returns true if successful, false if the deque is full

    SortedSet

    SortedSet is an interface that extends Set and adds additional methods to handle elements in sorted order.

    Classes of SortedSet: TreeSet

    Methods of SortedSet
    • first (E) --> returns the first element in the sorted set
    • last (E) --> returns the last element in the sorted set
    • headSet(E toElement) --> returns a sorted set containing all the elements in this sorted set less than the specified element
    • tailSet(E fromElement, boolean inclusive) --> returns a sorted set containing all the elements in this sorted set greater than or equal to the specified element
    • comparator() --> returns the comparator that defines the order of elements in this sorted set
    • subSet(E fromElement, E toElement) --> returns the set fromElement to toElement as new sorted set
    NavigableSet

    NavigableSet is an interface that extends SortedSet and adds additional methods to handle elements in a navigable order.

    Classes of NavigableSet: TreeSet

    Methods of NavigableSet
    • lower(E e) --> returns the greatest element in the sorted set less than or equal to the given element
    • floor(E e) --> returns the greatest element in the sorted set less than the given element
    • ceiling(E e) --> returns the least element in the sorted set greater than or equal to the given element
    • higher(E e) --> returns the least element in the sorted set greater than the given element
    • pollFirst() --> removes and returns the first element in the sorted set
    • pollLast() --> removes and returns the last element in the sorted set
    • descendingSet() --> returns a sorted set with the elements in reverse order
    • descendingIterator() --> returns an iterator over the elements in the sorted set in reverse order
    Map

    Map is an interface that represents a collection of key-value pairs.

    • Map doesn't allows duplicate keys but allows duplicate values.
    • Map allows heterogeneous elements (keys and values can be different data types).
    • Insertion order is preserved.
    • HashMap, LinkedHashMap allows null values any number of times but allows only one null key.
    • TreeSet, TreeMap allows null keys and values

    Classes of Map: HashMap, Hashtable, LinkedHashMap, TreeMap, ConcurrentHashMap, IdentityHashMap, WeakHashMap

    Methods of Map
    • put(K key, V value) --> adds the specified key-value pair to the map if the key is not already present
    • put(K key, V value) --> replaces the value for the specified key with the specified value if the key is already present
    • get(Object key) --> returns the value to which the specified key is mapped, or null if the map contains no mapping for the key
    • containsKey(Object key) --> checks whether the map contains the specified key
    • containsValue(Object value) --> checks whether the map contains the specified value
    • remove(Object key) --> removes the mapping for the specified key from the map if it is present
    • clear() --> removes all mappings from the map
    • size() --> returns the number of key-value mappings in the map
    • entrySet() --> returns a Set view of the mappings in the map
    • keySet() --> returns a Set view of the keys in the map
    • values() --> returns a Collection view of the values in the map
    • hashCode() --> returns the hash code value for this map
    • replace(k key, v value) --> replaces the value for the specified key with the specified value if the key is already present
    SortedMap

    SortedMap is an interface that extends Map and adds additional methods to handle elements in sorted order.

    Classes of SortedMap: TreeMap

    Methods of SortedMap
    • firstKey() --> returns the first (lowest) key in the sorted map
    • lastKey() --> returns the last (highest) key in the sorted map
    • headMap(K toKey) --> returns a sorted map containing all the mappings in this sorted map less than the specified key
    • tailMap(K fromKey, boolean inclusive) --> returns a sorted map containing all the mappings in this sorted map greater than or equal to the specified key
    • comparator() --> returns the comparator that defines the order of elements in this sorted map
    • subMap(K fromKey, K toKey) --> returns the map from fromKey to toKey as new sorted map
    NavigableMap

    NavigableMap is an interface that extends SortedMap and adds additional methods to handle elements in a navigable order.

    Classes of NavigableMap: TreeMap

    Methods of NavigableMap
    • headMap(k toKey, boolean inclusive) --> returns a value greater than the specified key.
    • tailMap(k fromKey, boolean inclusive) --> returns a value less than the specified key.
    • subMap(k fromKey, k toKey) --> returns a value between the specified keys.
    • lowerKey(k key) --> returns the greatest key less than or equal to the specified key.
    • floorKey(k key) --> returns the greatest key less than the specified key.
    • ceilingKey(k key) --> returns the least key greater than or equal to the specified key.
    • higherKey(k key) --> returns the least key greater than the specified key.
    • descendingMap() --> returns a reverse order view of the map.

    Comparator

    Methods in Comparator interface:

    Mutlitasking
    • Excecution several tasks simultaneously is the concept of multitasking.

    There are 2 types of multitasking

    • Process based Mutlitasking (PBM)
      • Executing several tasks simultaneously where each task is a separate independent programm is called PBM
      • These are OS level but not at programming level.
      • Ex: CPU, Browser
    • Thread based Multitasking
      • Executing several tasks simultaneously where each task is a separate independent part of the same program is called TBM and each independent part is called Thread.
      • TBM is best suitable at programmatic level.
      • Whether it is PB or TB the main advantage of multitasking is to reduce response time and to improve the performance of the system.

    The main important application areas of multithreading are

    • To develop multimedia graphics.
    • To implement viedo Animations
    • To develop viedo games
    • To develop a server
    • To develop server and web application servers

    Lambda Expressions

    • Lambda expressions allow us to write a single line of code in place of a method
    • Lambda expressions can be used as arguments to methods, constructors, and in most places where a functional interface is expected
    • Lambda expressions are anonymous, meaning they don't have a name
    Example:
    interface Interf1{
    void method1();
}

public class Test1 {
    Interf1 i1 = () -> {
        System.out.println("Hello World!")
    }
    i1.method1();
}
    Predicate Interface

    Predicate is a functional interface that takes one argument and returns a boolean value.

    Methods in Predicate interface:

    • test(T t) --> returns true if the given argument matches the predicate
    • negate() --> returns a predicate that negates the given predicate
    • and(Predicate other) --> returns a predicate that is true if both this predicate and the given predicate are true
    • or(Predicate other) --> returns a predicate that is true if either this predicate or the given predicate is true
    • isEqual(Object other) --> returns a predicate that is true if this predicate and the given object are equal
    Function Interface

    Function is a functional interface that takes one argument and returns a result.

    Methods in Function interface:

    • apply(T t) --> returns the result of applying this function to the given argument
    • andThen(Function after) --> returns a composed function that first applies this function to its input, and then applies the after function to the result
    • compose(Function before) --> returns a composed function that first applies the before function to its input, and then applies this function to the result
    • identity() --> returns a function that always returns its input argument
    Consumer Interface

    Consumer is a functional interface that takes one argument and returns nothing.

    Methods in Consumer interface:

    • accept(T t) --> takes a value and performs an action on it
    • andThen(Consumer after) --> returns a composed consumer that, when this consumer is applied to an input, applies the after consumer to the result
    Supplier Interface

    Supplier is a functional interface that won't take any argument andreturns a result.

    Methods in Supplier interface:

    • get() --> returns a result generated by this supplier
    • supplier(Supplier other) --> returns a composed supplier that, when this supplier is applied, applies the other supplier to the result
    BiPredicate Interface

    BiPredicate is a functional interface that takes two arguments and returns a boolean value.

    Methods in BiPredicate interface:

    • test(T t, U u) --> returns true if both the given arguments match the predicate
    • negate() --> returns a predicate that negates the given predicate
    • and(BiPredicate other) --> returns a predicate that is true if both this predicate and the given predicate are true
    • or(BiPredicate other) --> returns a predicate that is true if either this predicate or the given predicate is true
    • isEqual(Object other) --> returns a predicate that is true if this predicate and the given object are equal
    BiFunction Interface

    BiFunction is a functional interface that takes two arguments and returns a result.

    Methods in BiFunction interface:

    • apply(T t, U u) --> returns the result of applying this function to the given arguments
    • andThen(BiFunction after) --> returns a composed function that first applies this function to its input arguments, and then applies the after function to the result
    • compose(BiFunction before) --> returns a composed function that first applies the before function to its input arguments, and then applies this function to the result
    • identity() --> returns a function that always returns its input arguments

    Stream API

    • Stream is a sequence of elements that supports aggregate operations
    • Stream API provides a functional programming style for data processing
    • Stream API is implemented as a pipeline of intermediate and terminal operations

    Methods of stream API:

    • stream() --> returns a stream containing elements of the source collection
    • parallelStream() --> returns a parallel stream containing elements of the source collection
    • filter(Predicate predicate) --> returns a stream containing only the elements of this stream that satisfy the given predicate
    • map(Function mapper) --> returns a stream containing the results of applying the given function to the elements of this stream
    • flatMap(Function> mapper) --> returns a stream containing the results of applying the given function to the elements of this stream and concatenating the results
    • distinct() --> returns a stream containing only the distinct elements of this stream
    • sorted() --> returns a stream that is sorted according to the natural ordering of its elements
    • count --> returns the number of elements in the stream
    • forEach(Consumer action) --> performs an action for each element of the stream
    • reduce(T identity, BinaryOperator accumulator) --> returns the result of applying a binary operator to all elements in the stream, or identity if the stream is empty
    • toArray() --> returns an array containing all elements of the stream
    • collect(Collector collector) --> returns the result of applying a collector to the elements of the stream
    Optional Class

    Optional class is a container object that may or may not contain a non-null value.

    Methods of Optional class:

    • isPresent() --> returns true if this Optional object contains a value
    • get() --> returns the value of this Optional object if present, otherwise throws NoSuchElementException
    • orElse(T other) --> returns the value of this Optional object if present, otherwise returns the given other value
    • orElseGet(Supplier supplier) --> returns the value of this Optional object if present, otherwise returns the result of the given supplier
    • orElseThrow(Supplier supplier) --> throws the exception returned by the given supplier if this Optional object is empty
    • ifPresent(Consumer consumer) --> performs the given action if this Optional object is present
