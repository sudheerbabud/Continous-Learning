Introduction of PL/SQL

PL/SQL, which stands for Procedural Language extensions to the Structured Query Language (SQL).
It is a combination of SQL along with the procedural features of programming languages.
It was developed by Oracle Corporation in the early 90's to enhance the capabilities of SQL.
PL/SQL is one of three key programming languages embedded in the Oracle Database, along with SQL itself and Java.

Sub program

A subprogram is a program unit/module that performs a particular task. These subprograms are combined to form larger programs. This is basically called the 'Modular design'. A subprogram can be invoked by another subprogram or program which is called the calling program.

A sub program can be created:

  • At the schema level --> standalone subprogram
  • Inside a package level --> packaged subprogram
  • Inside a PL/SQl block
Parts of a subprogram
  • Declarative Part
  • Executable Part
  • Exception-handling

Modes of Procedures:

Creating a procedure
CREATE [OR REPLACE] PROCEDURE procedure_name 
[(parameter_name [IN | OUT | IN OUT] type [, ...])] 
{IS | AS} 
BEGIN 
< procedure_body > 
END procedure_name;
Example: IN
CREATE DEFINER=`root`@`localhost` PROCEDURE `new_procedure`(in x int)
BEGIN
    select * into en from emp where en0 =7766;
END
                            
Call `new_procedure`(x)

calling: call new_procedure(7839)

Example: OUT
CREATE DEFINER=`root`@`localhost` PROCEDURE `new_procedure`(out en int)
BEGIN
    select empno into en from emp where ename = 'king';
END
                            
Call `new_procedure`(@en)
Select @en;

calling: call new_procedure(@val); select @val;

Example: IN/OUT
CREATE DEFINER=`root`@`localhost` PROCEDURE `new_procedure`(in eno int, out en int)
BEGIN
    select sal into en from emp where empno= eno;
END
                            
Call `new_procedure`(7839, @en)
Select @en;

calling: call new_procedure(7839, @val); select @val;

Example: INOUT
CREATE DEFINER=`root`@`localhost` PROCEDURE `new_procedure`(inout result int)
BEGIN
    Set result = result +10;
END
                        
Set @value = 78;
Call `new_procedure`(@value)
Select @value;

calling: select @val = 10; call new_procedure(@val); select @val;

Functions

A function is same as a procedure except that it returns a value.

A function doesn't have modes for parameters. it must be deterministic.

Creating a function
CREATE FUNCTION function_name()
RETURN return_datatype 
BEGIN 
 < function_body > 
return 1;
END
Example:
CREATE DEFINER=`root`@`localhost` FUNCTION `fun3`(a int) RETURNS int
DETERMINISTIC
BEGIN
    set a = a + 10;
RETURN a;
END

calling: select deptno, fun3(10) from emp;

Triggers

Triggers are stored programs, which are automatically executed or fired when some events occur.

Triggers are two types. 1) row level triggers 2) statement level triggers.. In PLSQL we have only row level triggers.

Creating a trigger
$$ Delimiter
CREATE TRIGGER Trigger_name 
before/after insert/update/delete on table_name
for each row
BEGIN 
 < function_body > 
END $$
Example:

Before this we have to crete the table for trigger data.

CREATE TABLE Audit (
sno int auto_increment primary key,
deptno int,
currentdate date
);
$$ Delimiter
CREATE TRIGGER Trig 
before insert on deptno
for each row
BEGIN 
    insert into Audit (deptno, currentdate) values (NEW.deptno, current_date());
END $$

calling: we must insert the data in deptno table, then trigger will automatically insert the data in audit table.

Conditional Statements

  • if then
  • if then else
  • if then elsif else --> by using storage procedure
    • CREATE DEFINER=`root`@`localhost` PROCEDURE `ifelse1`(in d int, out x varchar(30))
BEGIN
    if d = 10 then
        select * from emp where deptno = d;
        select max(sal) into x from emp where deptno = d;
    else if d = 20 then
        select * from emp where deptno = d;
        select max(sal) into x from emp where deptno = d;
        else if d = 30 then
            select * from emp where deptno = d;
            select max(sal) into x from emp where deptno = d;
            else if d = 40 then
                set x = "No employess in this dept";
                else 
                    set x = "Invalid input";
                end if;
            end if;
        end if;
    end if;
END

    calling: call ifelse1(10,@val); select @val;

  • Switch case --> by using storage procedure
    • CREATE DEFINER=`root`@`localhost` PROCEDURE `case2`(in dno int, out info varchar(30))
BEGIN
    case dno 
        when 10 then
            select * from emp where deptno = dno;
            select max(sal) into info from emp where deptno = dno;
        when 20 then 
            select * from emp where deptno = dno;
            select max(sal) into info from emp where deptno = dno;
        when 30 then 
            select * from emp where deptno = dno;
            select max(sal) into info from emp where deptno = dno;
        when 40 then 
            select * from emp where deptno = dno;
            set info = "No employees in dept";
        else
            set info = "Invalid input"; 
    end case;
END

    calling: call case2(10,@val); select @val;

Loops

  • for loop --> by using storage procedure
    • CREATE DEFINER=`root`@`localhost` PROCEDURE `loop2`(in n int, out result int)
BEGIN
    declare sum int default 0;
    declare num int default 1;
    nat_sum:loop
        set sum =sum + num;
        set num = num + 1;
        if num = n then
            set result =  sum;
            leave nat_sum;
        end if;
    end loop;
END

    calling: call loop2(10,@val); select @val;

  • while --> by using function
    • CREATE DEFINER=`root`@`localhost` FUNCTION `fun5`(in n int, out result int) returns int
DETERMINISTIC
BEGIN
    declare sum int default 0;
    declare num int default 1;
    nat_sum:while num <= n do
        set sum =sum + num;
        set num = num + 1;
        end while;
    return sum;
    end loop;
END

    calling: call loop2(10,@val); select @val;

    In while the loop runs until we get false

  • repeat --> by using function
    • CREATE DEFINER=`root`@`localhost` FUNCTION `repet`(d int) RETURNS int
DETERMINISTIC
BEGIN
    declare num int default 100;
    declare sum int default 0;
    addd: repeat
        set sum = sum + num;
        set num = num + 1;
        until num>=d
    end repeat;
RETURN sum;
END

    Calling: select repet(120);

    In repeat the loop runs until we get true

Cursors

Cursors are used to handle and process rows retrieved from a query. They act as pointers that allow you to iterate over query result sets row by row.

CREATE DEFINER=`root`@`localhost` PROCEDURE `cursors`(out joblist varchar(300))
BEGIN
    declare completed boolean default false;
    declare j varchar(30) default "";
    declare cur cursor for select job from emp;
    declare continue handler for not found set completed = true;
    set joblist = '';
        open cur;
            fetch_joblist:loop
                fetch cur into j;
                if completed = true then
                    set joblist = concat(j,";",joblist);
                        leave fetch_joblist;
                    end if;
            end loop;
        close cur;
END

Calling: call cursors(@ha); select @ha;
